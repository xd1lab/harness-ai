Drive a session over REST/SSE (no client tooling needed):

{{- if .Values.ingress.enabled }}
  BASE=https://{{ .Values.ingress.host }}
{{- else }}
  kubectl -n {{ .Release.Namespace }} port-forward svc/{{ include "boltrope.fullname" . }}-orchestratord 8080:8080 &
  BASE=http://localhost:8080
{{- end }}
  TOKEN=<an RS256 access token from your IdP with the tenant_id claim>

  curl -sS -X POST "$BASE/v1/sessions" \
    -H "Authorization: Bearer $TOKEN" -H 'Content-Type: application/json' \
    -d '{"mode":"default"}'
  curl -N -X POST "$BASE/v1/sessions/<id>/run" \
    -H "Authorization: Bearer $TOKEN" -H 'Content-Type: application/json' \
    -d '{"text":"Reply with exactly: OK"}'

Remember to register each tenant UUID from your IdP's tenant_id claim in the
tenants table (grant.tenants, or by hand) — sessions are RLS-scoped to it.
